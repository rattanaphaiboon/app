📝 Design decisions (รออ่าน + comment)

  • Auto-default method: ถ้าคลังอื่นมีสต็อก → ✓ แนะนำ "โอนจาก" · ถ้าไม่มีเลย → ✓ แนะนำ "สั่งซื้อ" (ดู Scenario A vs B)
  • Toggle method ได้เสมอ: user เลือกได้ทั้ง 2 ทาง · ไม่ lock เหมือนเดิม
  • Header color เปลี่ยนตาม method: สีฟ้า=โอน · สีส้ม=สั่งซื้อ · ผู้ใช้รู้ทันทีว่าทำ action อะไรอยู่
  • Vendor + ETA auto: จาก CAT VENDOR + SKU_LEAD_TIME · read-only · ลด typo
  • "สั่งด่วน" = checkbox (ไม่แยก 3rd mode) · ปุ่ม "⚠️ ใกล้หมด" บนการ์ดเปิด modal นี้แต่ auto-check ให้ (Scenario C)
  • 3 ปุ่มบนการ์ดเหลือ 2: เอา "⚠️ ใกล้หมด>สั่งซื้อ" ออก (รวมมา modal นี้แล้ว) · เหลือ "🛒 ขอสินค้า" + "💰 ขอปรับราคา" + "📱 ส่งข้อมูล"
  • หรือเหลือ 2 ปุ่ม: "🛒 ขอสินค้า" + "📱 ส่งข้อมูล" · ปุ่ม "ขอปรับราคา" ก็เอาออก (รวมเข้า modal นี้?) — รอ Surat ตัดสิน
