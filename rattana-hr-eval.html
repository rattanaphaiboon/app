🔒 สิทธิ์การเข้าถึง (Access Control)

ระบบใช้ 2 ระดับ ในการกำหนดสิทธิ์ — EvalAccess (override) มาก่อน User Role (default)

1) Override: คอลัมน์ EvalAccess ใน Users sheet

ค่าความหมายคำพ้อง (ใส่แทนได้)
allเห็นข้อมูลทุกสาขาทั้งหมด, admin, full, any
warehouseเห็นเฉพาะสาขาตัวเองสาขา, W, dept
selfเห็นเฉพาะตัวเอง (match by รหัสพนักงาน)ตัวเอง, own, me
(เว้นว่าง)ใช้ default ตาม User Role-

2) Default: ตาม User Role (1-8)

Roleระดับสิทธิ์ดูประเมิน
7-8HR / Executive / Adminเห็นทุกสาขา
4-6Supervisor / Managerเห็นเฉพาะสาขาตัวเอง
1-3Staff / Viewerเห็นเฉพาะตัวเอง
💡 ตัวอย่างการใช้: ถ้า HR มี role=5 (ปกติเห็นแค่สาขาตัวเอง) แต่อยากให้เห็นทุกสาขา → ใส่ all ในคอลัมน์ EvalAccess
⚠️ การเพิ่มคอลัมน์: เพิ่มคอลัมน์ใหม่ชื่อ EvalAccess ในชีท Rattana Users for apps (ID: 1M6HdIS...) — ถ้ายังไม่เพิ่ม ระบบจะ fall back ไปใช้ User Role อัตโนมัติ ไม่ error
