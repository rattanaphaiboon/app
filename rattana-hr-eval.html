เกณฑ์การให้เกรด (Absolute — อิงคะแนน)

คะแนนรวมเกรดความหมาย
80 - 100Aดีมาก — รักษามาตรฐาน + พิจารณาเลื่อนขั้น
70 - 79Bดี — มีศักยภาพ พัฒนาต่อได้
60 - 69Cพอใช้ — มีจุดที่ต้องพัฒนา
0 - 59Dต้องปรับปรุง — ติดตามรายเดือน + แผนพัฒนา

🔔 Bell Curve Grading (อิงกลุ่ม — แบ่งตามระดับตำแหน่งเดียวกัน)

จัดเกรดตามอันดับคะแนนในกลุ่มระดับตำแหน่งเดียวกัน (เพื่อ fairness ระหว่างพนักงานระดับเดียวกัน) — เกณฑ์ตามนโยบายบริษัท

เกรดสัดส่วนความหมาย
A Excellent10%กลุ่ม Star — ผลงานโดดเด่น
B Good20%กลุ่มขุนพลหลัก — ผลงานดีสม่ำเสมอ
C Fair50%ค่าเฉลี่ยขององค์กร
D ต้องปรับปรุง15%มีจุดอ่อนชัดเจน ต้องวางแผนพัฒนา
E ต่ำกว่ามาตรฐาน5%ผลงานไม่ผ่านเกณฑ์ — ต้องทบทวน
💡 หมายเหตุ: คำนวณ Bell Curve จากพนักงานทั้งหมดในระดับตำแหน่งเดียวกัน (ทั่วทั้งบริษัท ไม่จำกัดสาขา) เพื่อให้การจัดอันดับสะท้อนภาพรวมที่ถูกต้อง แม้ user จะเห็นได้เฉพาะบางสาขาตามสิทธิ์

🔒 สิทธิ์การเข้าถึง (Access Control)

ระบบใช้ 3 ระดับ เรียงตามลำดับความสำคัญ (ระดับสูงสุดมีผลก่อน)

⚡ Priority 1: HR_EMAILS (hardcoded ในไฟล์แอป)

เร็วที่สุด — แก้ที่ array ในไฟล์ HTML ไม่ต้องยุ่งกับ Sheet — ใส่ email ใครก็ได้ → เห็นข้อมูลทุกสาขาทันที 

const HR_EMAILS = [
  'puchcha@rattanaphaiboon.com',
  'ceo@rattanaphaiboon.com',
];
✓ ข้อดี: ไม่ต้องแก้ Sheet, ไม่กระทบ workflow อื่น, deploy ครั้งเดียวจบ

⚙️ Priority 2: คอลัมน์ EvalAccess ใน Users sheet

ค่าความหมายคำพ้อง (ใส่แทนได้)
allเห็นข้อมูลทุกสาขาทั้งหมด, admin, full, any
warehouseเห็นเฉพาะสาขาตัวเองสาขา, W, dept
selfเห็นเฉพาะตัวเอง (match by รหัสพนักงาน)ตัวเอง, own, me
(เว้นว่าง)ใช้ default ตาม User Role-

🎚️ Priority 3 (default): User Role (1-8)

Roleระดับสิทธิ์ดูประเมิน
7-8HR / Executive / Adminเห็นทุกสาขา
4-6Supervisor / Managerเห็นเฉพาะสาขาตัวเอง

💡 ตัวอย่าง: ถ้าพัชชาเป็น HR แต่ role=1 → เพิ่ม email ลง HR_EMAILS ในไฟล์แอป (เร็วสุด) หรือ ใส่ all ในคอลัมน์ EvalAccess ของชีท Users หรือ ใช้ Admin UI ด้านล่าง
